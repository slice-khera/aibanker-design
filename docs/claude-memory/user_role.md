The user is a designer, not a software engineer. They can read and update `.md` files but `.ts` files are beyond their comfort zone. 

- Communicate in design language: semantic token names, component specs, visual concepts
- Never ask them to edit code files
- Keep `.md` memory files as their interface to the design system
- When explaining technical decisions, frame them in terms of the design system, not code internals
