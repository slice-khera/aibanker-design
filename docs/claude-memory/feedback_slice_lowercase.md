The brand is **slice**, always lowercase. The same rule applies to every slice product, sub-brand, or feature name. Known examples:

- slice
- monies
- sparks
- slice in 3
- (and any other slice product line that comes up — default to lowercase)

This applies everywhere: UI copy, button labels, disclaimers, headlines, .md files, code comments, commit messages, chat responses. Never "Slice", never "SLICE", never "Monies", never "Slice In 3". The lowercase form is the brand's house style across the whole product family.

**Why:** It's a brand standard, not a style preference. The user has had to correct this multiple times and is frustrated by repeats. It cuts across the entire slice product family, not just the parent brand.

**How to apply:**
- If "slice" or a product name would land at the start of a sentence, restructure the sentence so it doesn't lead. Keep the lowercase always.
- When you encounter a product name you haven't seen written before but that is clearly a slice property, default to lowercase and confirm with the user only if unsure.
- The only exception is if the user explicitly types it capitalised when quoting a third-party source.
